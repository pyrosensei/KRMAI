
Summary
-------
KRMAI is a local-first, AI-powered chatbot for KR Mangalam University. It provides instant, document-backed answers to student questions about university policies, fees, admissions, placements, hostels, and more. The system ingests official documents (TXT, PDF, DOCX), converts them into vector embeddings, and uses a local LLM (Ollama) to generate conversational answers grounded in retrieved document chunks. Every answer includes source citations for trust and transparency. The app is fully private (no cloud APIs), supports informal language, and is optimized for fast, reliable campus use.

Backend Internals (How It Works Exactly)

rag_engine.py:
- Sets offline defaults:
	- HF_HUB_OFFLINE=1
	- TRANSFORMERS_OFFLINE=1
- Loads Chroma from chroma_db/
- Creates retriever with k=3
- Checks Ollama at http://localhost:11434/api/tags
- Uses OllamaLLM(model="llama3.2:3b", timeout=300, num_predict=512)
- Has slang expansion map (common abbreviations + college-specific terms)
- Builds prompt with:
	- language/tone rules (English/Hindi/Hinglish behavior)
	- strict "use context only"
	- fallback string if not enough info
- Maintains short chat memory:
	- max_history = 6 messages
- Query flow:
	- expand slang
	- retrieve docs
	- format prompt with context + history
	- invoke llm
	- return {answer, source_documents}

api.py:
- FastAPI lifespan initializes one global RAGEngine
- CORS configured allow_origins=["*"], allow_credentials=False
- GET /health: returns rag_engine.status
- POST /chat:
	- request body: message and optional history
	- forwards to rag_engine.query
	- deduplicates source docs by source-page
	- response model: answer + sources[]

ingest.py:
- Loads .pdf, .docx, .txt using LangChain loaders
- Splits chunks with:
	- chunk_size=1000
	- chunk_overlap=200
- Builds embeddings with all-MiniLM-L6-v2
- Wipes existing chroma_db each run (shutil.rmtree) for clean rebuild
- Persists vector store to chroma_db/

app.py (Streamlit):
- Keeps message history in st.session_state
- Loads RAGEngine() once into session state
- Chat UI with source listing
- "Reload Engine" button to recreate engine

Scripts And Operational Behavior

start.sh:
- Checks ollama binary
- Starts ollama serve if needed
- Auto pulls llama3.2:3b if missing
- Auto-installs python deps if FastAPI missing
- Auto-runs ingestion if chroma_db empty
- Default mode starts backend + frontend
- Optional mode streamlit launches Streamlit app

start.bat:
- Checks python
- Creates/activates venv
- Installs requirements
- Menu:
	- ingest
	- run streamlit
	- exit

Potential Weak Spots / Risks

- ACTIVE_SESSION_KEY constant exists in App.jsx but is not used.
- CORS currently allows all origins (*) for local dev convenience.
- start.sh uses python/pip assumptions; if shell path differs, env mismatch can happen.
- Ingestion wipes prior Chroma DB every run by design.
- No auth/rate-limit/user isolation.
- Chat memory is in-process and ephemeral (resets on backend restart).
